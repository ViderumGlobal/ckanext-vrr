API OpenVRR

Für die kann die folgende URL zu unserem Fahrgastinformationssystem genutzt werden: XML-Schnittstelle

Wir halten uns frei diesen Zugriff auch für interne Tests zu benutzen, so dass es zeitweise nicht zu plausiblen Ergebnissen kommen kann. Hier eine Beispielanfrage: XML-Schnittstelle

In der Schnittstellendokumentation werden alle HTTP-Parameter zur Nutzung für folgende Anfragen beschrieben:

  • StopFinder-Request: Punkt- bzw. Haltestellen-Suche
  • Trip-Request: Anfrage von Fahrtauskünften
  • PS-Request: Anfrage eines Pendlerfahrplans
  • DM-Request: Anfrage eine Abfahrtsmonitors
  • DMTTP-Request: Anfrage des Soll-Abfahrtsmonitors
  • StopTimetable-Request: Anfrage eines Aushangfahrplans für eine Linie
  • StopList-Request: Anfrage aller Haltestellen eines Verbundsgebiets
  • SystemInfo-Request: Anfrage von Systeminformation
  • AddInfo-Request: Anfrage aktueller Meldungen
  • Map-Request: Anfrage von Karten

Gegenwärtig ertüchtigen wir unsere Systemarchitektur, um die EFA-XML-Schnittstelle im Rahmen von OpenVRR für den produktiven Betrieb bereitstellen zu können. Daher darf diese Schnittstelle zum Fahrgastinformationssystem darf nicht für den produktiven Betrieb verwendet werden.

Sollten die EFA-XML-Schnittstelle bereits jetzt für den produktiven Betrieb benötigt werden, dann bitten um Kontaktaufnahme per Mail.
